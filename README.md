# サプリ摂取トラッカー（PWA）

毎日のサプリ摂取をチェックで記録し続ける、**端末内完結・記録専用**のアプリ。
医療的な推奨・警告ロジックは持たない（純粋な記録ツール）。元の「サプリ・スケジュール（1枚HTML）」をインタラクティブ＋永続化したもの。

## ファイル構成
| ファイル | 役割 |
|---|---|
| `index.html` | 画面・スタイル（元HTMLの和風トーンを踏襲） |
| `app.js` | 全ロジック（記録・集計・編集・入出力） |
| `manifest.webmanifest` | PWA定義（ホーム追加・アイコン） |
| `sw.js` | Service Worker（オフライン起動） |
| `icon-192.png` / `icon-512.png` / `icon-maskable-512.png` | アイコン |

## できること
- **今日**：時間帯（朝〜寝る前）ごとのチェックリスト。タップで記録（タイムスタンプも保存）。上部に達成率リング・連続達成日数。
- **達成率**：必須項目のうちチェックした割合。`任意`項目（人参ジュース・夜の消化酵素など）は分母に含めない＝飲まなくても達成率は下がらない。
- **日次リセット**：日付が変わると今日の画面はリセット。前日までの記録は履歴に残る。境界時刻は設定で変更可（標準0時／就寝が遅ければ5時など）。
- **履歴**：月カレンダーのヒートマップ（色が濃いほど達成）。日をタップすると内訳を確認・**後から修正**（飲み忘れ・付け忘れの訂正）。日ごとのメモも可。
- **継続の記録**：現在の連続達成・最長連続・記録日数・達成日数。
- **項目別の遵守率**：直近30日で各項目を飲めた割合。
- **スケジュール編集**：項目の追加・編集・無効化・並べ替え・用量変更。変更は**今日以降**に適用し、過去の記録は当時の内容のまま残る（バージョン管理）。
- **データ入出力**：JSON（全データのバックアップ／復元）・CSV（`date,itemId,name,checked,timestamp`／検査値との突き合わせ用）。

## データの保存場所
- すべて端末内の **IndexedDB**（`supplement-tracker`）に保存。サーバー送信なし・アカウント不要。
- バックアップは設定 → データ → 「JSONで書き出し」。機種変更時は書き出したJSONを「JSONから復元」で戻す。

## iPhoneで使う（ホーム画面に追加）
上の公開URLを iPhone の **Safari** で開く → 共有ボタン → **「ホーム画面に追加」** → ホームのアイコンから全画面・オフライン起動。

（自前で別配信する場合）PWAのインストールには **HTTPS配信**が必要（iOSの仕様）。次のどれかで配信する：

1. **無料の静的ホスティング**（推奨・最短）
   このフォルダの中身をそのままアップロードするだけ：
   - Netlify Drop（https://app.netlify.com/drop にフォルダをドラッグ）
   - Cloudflare Pages / GitHub Pages など
2. 配信URLを iPhone の **Safari** で開く → 共有ボタン → **「ホーム画面に追加」**
3. ホームのアイコンから起動（全画面・オフライン動作）

> Macでの動作確認だけなら `python3 -m http.server` でローカル起動でOK（`localhost` はSWが動く例外）。
> file:// で直接開くと Service Worker が動かず、ホーム追加・オフラインは使えない（中身の確認は可能）。

## 更新するとき
`app.js` や `index.html` を変えたら、`sw.js` の `CACHE = 'suppl-tracker-v1'` の版番号を上げる（例：`-v2`）。
これで古いキャッシュが破棄され、次回起動時に新しい内容が反映される。
